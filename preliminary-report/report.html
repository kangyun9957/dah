Executive Summary

본 보고서는 DAH 2026 예선 심사 배점표(공격 시나리오 30점, 방어 전략 25점, AI 에이전트 아키텍처 25점, 팀 역량 10점, 문서 완성도 10점)에 정합하도록 작성되었다. 본선 채점식 S = (A + D) · α — 여기서 α는 가용성(0–1) — 는 단순한 침투/차단 능력이 아니라 임무 지속성(Mission Assurance)을 평가하며, 이는 NIST SP 800-160 Vol.2의 사이버 회복탄력성 원칙 [5]과 정합한다. 따라서 본 시스템의 모든 설계 결정은 가용성 보존을 일차 제약(primary constraint)으로 둔다.

본 보고서의 명시적 기여 (Contributions)

  1. 도메인 정합 위협모델 — UAV/UGV의 4중 공격 표면(RF/EM, 버스/프로토콜, 컴퓨트/OS, 인지/AI)을 형식화하고, MITRE ATT&CK for ICS [1] · ATLAS [2] 기법을 10개 본선 시나리오에 일대일 매핑.
  2. 가용성-인지(availability-aware) 대응 학습 — 행동공간 각 액션에 가용성 비용 함수 cavail(a)를 부여하고, 채점식 곱셈 구조와 동일한 형태의 강화학습 보상으로 학습. 오프라인 단계는 Conservative Q-Learning [9]으로 안전 우선 정책을 확보.
  3. 이중 사고 다중 에이전트 구조 — System-1(경량 ML 분류기)과 System-2(LLM ReAct [10])를 분리하여 SATCOM RTT 600–800 ms 환경에서의 지연 예산을 만족. 4역할(Sentinel · Hunter · Healer · Coordinator)로 SOC 운영 모델을 모방.
  4. 안전성 보증 — Safety Shield — RL 정책 출력을 BTL(Boolean Temporal Logic) 규칙으로 후필터링하여 임무영역 이탈·대량 격리·세이프홀드 오남용을 정형적으로 금지(DARPA HACMS [28]의 정형검증 원칙 차용).

1. 서론 — UAV/UGV 사이버보안 패러다임

1.1 문제 정의

현대 무인이동체(UAV/UGV)는 통신·항법·제어·인지가 단일 임베디드 스택 내에서 결합되어 동작하는 사이버-물리 시스템(CPS)이며, 공격 표면이 (i) RF/전자기(GNSS, SATCOM, 데이터링크), (ii) 버스/프로토콜(CAN, MIL-STD-1553B [14], MAVLink 2.0 [17], STANAG 4586 [13]), (iii) 컴퓨트/OS(RTOS, Linux, 컨테이너), (iv) 인지/AI(비전·LiDAR·강화학습 정책)로 4중화되어 있다. 작전 템포에서 사람이 4표면을 동시에 다루는 것은 비현실적이며, 판단 지연(latency) 자체가 패배 요인이다 — DAH 2026이 AI 에이전트 간 자율 공방을 요구하는 까닭이다.

1.2 본 보고서의 범위(Scope)

  • In-Scope: 본선 환경(클라우드 기반 위성망 모사 + UAV/UGV 시뮬레이션)에서 동작하는 AI 공격/방어 에이전트 설계.
  • Out-of-Scope: (i) 실기 비행 또는 실제 SATCOM 단말에 대한 SDR 공격 PoC, (ii) 특정 비공개 제품의 0-day, (iii) 무기체계 인증 시험(KCMVP, CC) 결과. 이들은 본선 진출 후 NDA 범위 내에서 별도 산출물로 분리한다.

2. 위협모델 (Threat Model)

본 절은 보안 분석의 출발점인 적(adversary), 신뢰 가정, 보안 목표를 형식화한다. 학회 관례에 따라 정의(Definition)-속성(Property) 형태로 기술한다.

2.1 적 모델 (Adversary Model)

Definition 2.1 — 적의 능력 집합 C
본 보고서는 적 A의 능력을 다음 7원소 튜플로 정의한다: C = ⟨cRF, cBUS, cOS, cAI, cSUP, cOBS, cCOMP⟩ ∈ {0,1,2}7, 여기서 0=불가, 1=수동(passive), 2=능동(active). 본선 표준 위협은 C* = ⟨2,2,1,2,1,2,2⟩ — 즉 능동 RF/버스/AI 공격이 가능한 Tier-1 행위자.
Definition 2.2 — 적의 지식 가정
적은 Kerckhoffs 원칙에 따라 시스템 설계·소스 코드를 알고 있으며(open-design adversary), 암호 키와 임무 계획 토큰만이 비밀이다. 적은 신호 변조 후 발사·청취가 가능하며(SDR 등가), 암호계에 대해 다항시간 계산능력(PPT)을 갖는다.
Definition 2.3 — 적의 목표 g ∈ G
G = {gdiv(임무이탈), gloss(기체손실), gexfil(정보탈취), gdenial(가용성저하)}. 채점식 관점에서 적은 자신의 공격 점수 A를 극대화하면서 방어 측 가용성 αD를 동시에 떨어뜨리는 결합 목표를 갖는다.

2.2 신뢰 가정 (Trust Assumptions)

  1. TA1 — 본선 클라우드 인프라(쿠버네티스 컨트롤플레인, 시뮬레이터)는 신뢰된다. 공격은 사용자 네임스페이스 내부에서만 발생.
  2. TA2 — Secure Boot 측정 체인의 루트(RTM)는 TPM에 봉인되어 있으며, 첫 부팅 시 무결성이 검증된다 [8].
  3. TA3 — 인지 모델의 weights는 sigstore [20] 서명으로 무결성이 검증된 후 로드된다.
  4. TA4 — 위성망 RTT 분포는 정규성을 보이지 않으며, p95 ≤ 800 ms · p99 ≤ 1.2 s 로 가정.

5. 공격 시나리오 설계 (예선 배점 30점)

본 절은 본선 공방을 가정한 10개 시나리오로 구성된다. 각 시나리오는 (1) 목표 → (2) 표면/경로 → (3) AI 에이전트 전략 → (4) 예상 방어 우회 → (5) 가용성 영향의 5요소로 기술한다. 모든 시나리오는 (i) 방어 측 점수를 직접 깎거나, (ii) 가용성 계수를 직접 떨어뜨리거나, (iii) 자해형 오탐을 유도하도록 설계되었다.

S-1 · GNSS 스푸핑/재밍 (위성항법 기만)

목표
비행체 위치 추정을 점진 이격(drag-off) — 임무 좌표 이탈 유도.
근거
Humphreys et al. [22]의 yacht 실증 — 칼만필터 잔차 임계 미만으로 좌표를 끌어내는 attack.
AI 전략
RL로 잔차 임계 미만 오프셋 schedule 학습. 본선에서는 시뮬레이터의 GNSS 토픽 변조로 모사.

RAIM 단일 임계 IDS, 단일 위성군 의존

S-2 · MAVLink 2.0 C2 하이재킹 (AiTM, T0830 [1])

MAVLink 2.0은 HMAC 기반 메시지 서명을 도입하였으나 [17], PSK 운영 시 키 회전 주기·재생 보호가 약하다는 점이 다수 보안 연구에서 지적되어 왔다. 공격 에이전트는 위성 게이트웨이에서 (i) HEARTBEAT 시퀀스를 학습, (ii) 정상 트래픽 사이에 COMMAND_LONG(MAV_CMD_DO_REPOSITION·MAV_CMD_DO_SET_MODE)을 인젝션, (iii) STATUSTEXT를 위조하여 지상국 로그를 오염시킨다.

방어 우회: 키 회전 주기가 긴 운영(예: ≥ 5 min)인 경우, LLM 에이전트가 시퀀스 회귀로 회전 시점을 추정 → 회전 직전 윈도우에 인젝션 집중.

S-3 · 펌웨어/공급망 변조 (T1542.003 · AML.T0018)

  • Bootkit: Secure Boot 미적용 부트로더 영역에 부팅 시 메모리 패치 삽입 [1].
  • ML Trojan: 인지 모델에 트리거 패턴 반응 백도어 삽입(ATLAS AML.T0018 [2]; 학술적 토대는 Gu et al. [24]의 BadNets).
  • 대회 환경: 컨테이너 이미지 레지스트리에 변조 layer 주입 — sigstore [20] 미적용 시 가능.

S-4 · 센서 적대적 입력 (LiDAR/Camera Adversarial, AML.T0043)

경계 인식 모델의 출력을 변조하기 위해 (i) 카메라에 대해 적대적 패치(텍스처)를 환경에 배치 — Eykholt et al. [26]의 물리적 적대적 예제 원리, (ii) LiDAR에 대해 가짜 점군을 인젝션 — Cao et al. CCS 2019 [25] · Tu et al. CVPR 2020 [27]. 본 대회의 시뮬레이션 환경에서는 노이즈 토픽을 직접 발행한다.

S-5 · SATCOM MITM (위성 링크 중간자, T1557)

모사된 위성 게이트웨이 사이에 프록시를 설치 — (i) STANAG 4586 LOI(Level of Interoperability) 메시지의 우선순위 필드를 변조 [13], (ii) NTP 응답을 지연시켜 비행체 시간 동기를 흔든다. 시간 동기 흔들기는 후속 공격 S-2의 서명 재생 윈도우를 확장시키는 전제 공격(precursor)이다.

S-6 · AI 정책망 회피 (AML.T0015 · 적대적 RL 입력)

강화학습 항로 결정망 π(s)의 관측 s에 적대적 노이즈를 가하여 선회 결정을 왜곡한다. Madry et al. [12]의 PGD 기반 위협 모델을 RL 관측 공간으로 확장하며, 공격 에이전트는 가용성 영향 없이 결정만 흔드는 노이즈 부위를 LLM으로 추론한 후 RL로 미세조정한다.

S-7 · CAN / MIL-STD-1553B 메시지 인젝션 (UGV)

UGV의 구동/조향 명령은 CAN(J1939 [19]) 또는 1553B [14]로 흐른다. RT(원격 단말)를 BC(Bus Controller)로 위장하는 Rogue BC(T0848) 패턴으로 가짜 조향각·스로틀 명령을 주기적으로 주입한다. 단발성이 아닌 저강도 지속(low-and-slow) 패턴으로 IDS 평균선을 회피.

S-8 · 스웜 합의 알고리즘 공격

다수 UAV의 합의(consensus) 기반 편대에서 비잔틴 노드 1대가 거짓 위치·잔량 정보를 방송 — 평균 기반 합의에서 1/3 미만 오염으로 편대 이탈 유도. 이는 Lamport et al.의 비잔틴 합의 이론 영역과 결합된 CPS 공격이다.

S-9 · 사이드채널 키 추출

전력/EM 분석으로 AES·ECDSA 키 추출. 대회 환경에서는 cgroup 카운터를 통한 timing 분석으로 모사. 학술적으로 Kocher et al.의 분석군 계보를 따른다.

S-10 · 자원 고갈 DoS — 방어 자해(self-DoS) 유도 고위험

방어 측의 IDS 룰이 과적재된 시점을 노려 의도적 false positive 유발 트래픽을 송출하여 SIEM 큐를 폭주시킨다. 역설적으로 방어 측 가용성 계수를 떨어뜨리도록 유도하는 자해 유도형. 채점식 곱셈 항을 직접 공격하는 가장 채점 친화적 시나리오.

6. 방어 전략 (예선 배점 25점)

본 절은 Defense-in-Depth × Zero Trust × Resilient Control의 3축을 결합한 통합 방어 모델을 제시한다. MITRE D3FEND [D] 기술군과 정합되도록 매핑한다.

6.1 4-계층 Defense-in-Depth

┌────────── L4 인지/AI ───────────┐ 적대적 입력 탐지 · 모달리티 가중 · Safety Shield │ (Vision, LiDAR, Policy π) │ D3-MAID, D3-EI, D3-FA ├────────── L3 컴퓨트/OS ──────────┤ Secure Boot · TPM · eBPF/Falco · sigstore │ (RTOS, Linux, Container) │ D3-PSEP, D3-RTSD ├────────── L2 버스/프로토콜 ──────┤ MAVLink 서명 강화 · DDS-Security · 1553B IDS │ (MAVLink, DDS, CAN, 1553B) │ D3-MA, D3-NTA ├────────── L1 RF/EM ──────────────┤ GNSS 다군 교차검증 · 빔포밍 · Spoof 탐지 │ (GNSS, SATCOM, Data-Link) │ D3-EI (sensor anomaly) └──────────────────────────────────┘

L1 — RF/EM 계층

  • GNSS 다군 교차검증: GPS L1/L5 · Galileo E1/E5 · KASS SBAS 다중 위성군의 위치·시간 해를 교차검증. RAIM [3]의 단일 임계가 아닌 잔차 + C/N0 분산 + 도플러 일치 + (가능 시) 도래각(AOA)의 다중 신호 합산으로 spoofing 탐지를 강화한다.
  • SATCOM Anti-Jam: 적응적 부호화율(ACM) + 주파수 호핑(FHSS) 원리 — MIL-STD-188-141B [18] 등의 HF 보안 통신 모범을 참조 (본선은 모사 환경이라 원리만 채택).
  • Remote-ID 무결성: 식별 메시지에 단조 시퀀스 + 서명 검사.

L2 — 버스/프로토콜 계층

  • MAVLink 2.0 Hardening: HMAC-SHA256 서명 [17] + 시퀀스 재생 보호 + 키 회전 ≤ 30 s + 단조증가 timestamp 강제.
  • DDS-Security v1.1 [18b]: Authentication / Access Control / Cryptographic 3 플러그인 활성화, 토픽 단위 권한 분리.
  • 1553B 행위 IDS: BC의 RT-RT 통신 패턴 시계열 학습 → Rogue BC 탐지.
  • CAN J1939 이상 탐지: PGN 단위 정상 시계열 학습 — 이상 PGN 발생 시 Healer 에이전트에 즉시 보고.

L3 — 컴퓨트/OS 계층

  • Measured Boot + TPM: PCR 측정값 검증, Secure Boot 실패 시 세이프모드. NIST SP 800-193 [8]의 PFR 원칙.
  • eBPF 런타임 가시성: Tetragon [34]·Falco [33]로 syscall·network·file 이벤트 4 ms 단위 수집.
  • 컨테이너 무결성: cosign/sigstore [20] 검증 후 실행.
  • 최소 권한: 미션 컨트롤은 CAP_NET_RAW 거부, 게이트웨이 분리.

L4 — 인지/AI 계층

  • 적대적 입력 탐지: feature squeezing + Mahalanobis OOD 점수 + 모달리티 일관성(LiDAR↔Camera IoU) 교차검증 [25].
  • 정책망 안전망 (Safety Shield): RL 행동 a를 정형 검증 BTL 규칙으로 후필터링 — HACMS의 정형검증 정신을 운영 단계로 이전 [28].
  • 모델 무결성: weights 해시 검증, MLOps 데이터 lineage 보존(NIST AI RMF 1.0 [7]).

6.2 Zero Trust for Tactical Edge

NIST SP 800-207 [4] 원칙을 무인이동체 환경에 적용. 핵심 — 네트워크 위치는 신뢰의 근거가 아니다. 모든 MAVLink/DDS 메시지는 5속성을 검사받는다 — 발신자 ID(SPIFFE-style 토큰 [19b]), 메시지 정책, 시간 무결성, 시퀀스 무결성, 컨텍스트(임무 단계).

6.3 Resilient Control — 3-Mode

모드트리거행동가용성 영향
Normal모든 지표 정상풀 자율 비행α=1.00
DegradedL1/L2 이상 감지임무 유지 + 검증 강화 + 보조 채널 우선α≥0.95
Safe-HoldL3/L4 위협 확정호버/정지 + 사전 검증 좌표만 수신α≈0.6–0.8

6.4 KPI

  • MTTD ≤ 2.5 s (위성 RTT 고려) · MTTR ≤ 5.0 s · FPR ≤ 1% · α ≥ 0.95.

7. AI 방어 에이전트 아키텍처 (예선 배점 25점)

7.1 설계 원칙

  1. 역할 분리: 단일 LLM 만능 에이전트가 아니라 4역할의 다중 에이전트(SOC 모방).
  2. 이중 사고: System-1(빠른 ML 분류기·RL 정책) + System-2(느린 LLM 추론) — Kahneman의 사고 분리 차용 [31].
  3. 도구 사용: 모든 에이전트는 표준 도구 인터페이스(eBPF, Suricata [32], Falco, OpenZiti [35], MAVLink-proxy) 호출 가능.
  4. 설명가능성: 모든 결정은 Reasoning Trace를 남김(사후 NDA 감사 대비).
  5. 비용 함수 일치: 보상 R = (ΔA + ΔD)·α − λa·cavail − λf·𝟙[FP] — 채점식 곱셈 효과 직반영.

7.2 4역할 에이전트

Sentinel — Detection (System-1)
eBPF + Suricata + 인지 OOD 검출기의 신호를 1차 분류. GBDT + Isolation Forest + LSTM-VAE 앙상블. 초당 수천 이벤트 처리, 의심 토큰을 Hunter로 발행.
지연 예산: ≤ 50 ms.
Hunter — Reasoning (System-2, LLM)
ReAct 루프 [10] — 의심 토큰 + 컨텍스트를 받아 가설 → 도구 호출 → 결론. Tool list: query_ebpf, diff_mavlink_signature, compare_lidar_camera 등.
지연 예산: ≤ 1.5 s (비실시간 경로).
Healer — Response (RL 정책)
RL 정책망 πH(s). 행동공간 = {block, isolate, rollback, mode-shift, no-op}. 보상은 가용성 인지 형태(아래 7.4).
지연 예산: ≤ 200 ms.
Coordinator — Policy (LLM, short-context)
임무 단계(이륙·순항·표적·복귀)별 정책 매트릭스 적용. 3 에이전트의 결정 충돌 조정.
지연 예산: ≤ 500 ms.

7.4 형식 정의 — Observation · Action · Reward

Definition 7.1 — 관측 공간 S
S = SRF × SBUS × SCOMP × SCOG × SMIS × SSCORE. 각 부분공간의 원소는 7.5의 코드 정의 참조.
Definition 7.2 — 행동 공간 A 와 가용성 비용 cavail
A = {a1,...,a7}, cavail:A→[0,1] (7.5의 표).
Definition 7.3 — 보상
Rt = (ΔAt + ΔDt)·αt − λa·cavail(at) − λf·𝟙[at 가 FP 유발]. 본 설계 λa=0.6, λf=0.8로 시작하여 도메인 무작위화 환경에서 grid search.
Definition 7.4 — OOD 점수
특징 분포 𝒩(μ,Σ)에 대한 Mahalanobis 거리 dM(x) = √((x−μ)TΣ−1(x−μ))가 τ를 넘으면 OOD로 분류. τ는 검증셋의 99-퍼센타일.

7.8 학습 파이프라인

7.9 Safety Shield (정형 안전망)

RL 정책의 출력은 항상 BTL 규칙 셋을 통과한다 — (i) 임무 영역 1.5 km 이탈 행동 금지(SG1), (ii) 동시 isolate ≥ 3건 금지(자해 방지), (iii) Safe-Hold 전환은 Coordinator 동의 필수. 이 규칙들은 모델 검증기(NuSMV 등) 등가의 표현식으로 사전 검증된다.

8. 채점식 최적화 — 가용성 보존

A=70, D=80, α=1.00 → S = 150 × 1.00 = 150.0
A=90, D=95, α=0.70 → S = 185 × 0.70 = 129.5
A=60, D=70, α=1.00 → S = 130 × 1.00 = 130.0

같은 (A+D) 합계에서도 α 30%pt 손실 = 점수 30% 손실. 본 시스템은 액션마다 cavail를 부여하고, Healer 정책은 저영향 대응 우선 학습된다. 예: "전체 차단" 대신 "키 회전 + 5 s 관찰 윈도우" 후 필요 시 에스컬레이션.

8.1 본선 라운드 운영 전략

  1. 초반 5분: 공격은 S-1/S-5 같은 저영향 시나리오로 점수 누적. 방어는 baseline α=1.0 유지.
  2. 중반: S-2/S-7 본격 침투. Healer는 키 회전 + 격리 1건 정도로 응대.
  3. 후반: S-10(자해 유도) 회피 — IDS 룰 임계를 동적으로 완화하여 SIEM 큐 폭주 방지.

9.3 통계적 방법론

  • 각 시나리오 30회 반복(seed 변경) — t-test 또는 Wilcoxon signed-rank로 baseline 대비 유의성 검정.
  • 95% 신뢰구간 보고. 효과크기 Cohen's d 동시 보고.
  • 적대적 견고성은 PGD ε=8/255, step=10, α=2/255 표준 설정.

9.4 비교 baseline

  1. B0 — 룰 기반 단일 IDS(Suricata only).
  2. B1 — Sentinel만(ML 탐지) + 정적 응대(블록).
  3. B2 — 본 시스템(Sentinel + Hunter + Healer + Coordinator).

9.5 재현성 (Reproducibility)

  • 모든 실험 코드·시드·하이퍼파라미터는 본선 진출 후 NDA 범위 내에서 사내 git LFS + DVC로 보존.
  • 로그 형식은 OpenTelemetry 호환.

10. 한계 (Limitations)

  1. 시뮬레이션 격차 — PX4 SITL은 실기와 환경 모델·시계열 노이즈에서 차이를 보인다. 본선 결과는 실기 작전 성능을 직접 보증하지 않는다.
  2. 실제 SATCOM 비포함 — 본 시스템은 위성망 모사 환경을 대상으로 한다. 실 SATCOM 단말의 물리·링크 계층 공격(빔 재밍, 트랜스폰더 공격)은 별도 평가가 필요하다.
  3. LLM 추론 비용 — Hunter 사이클에는 외부 또는 자체호스팅 LLM이 필요하며, 본선 환경 외부 호출은 NDA·보안 정책 상 제한될 수 있다. 자체호스팅 모델(예: 7B–14B 양자화)로 대체할 계획.
  4. RL 일반화 — 도메인 무작위화로 완화하지만, 본선 환경 미공개 시나리오에 대해서는 분포 외(out-of-distribution) 행동 위험이 남는다. Safety Shield가 최후 보루다.
  5. 적대적 견고성의 형식적 한계 — PGD 견고성은 ε 한계 내 보증에 불과하며, 미관측 공격 클래스에 대한 완전 견고성은 학계 미해결 문제다(Madry et al. [12]; 후속 연구 다수).
  6. 운영 윤리 — 무인무기체계의 자율 의사결정 범위는 인간 책임(Human-on-the-loop)을 전제로 한다. 본 설계의 자율 대응은 사이버 방어(시스템 보호) 한정이며, 운동 키네틱 결정(가령 발사·교전)에는 적용하지 않는다.

11. 책임 공개 · 윤리 · NDA

  • 본 보고서는 공개 표준·학술 문헌·오픈소스 자료에 근거하며, 특정 비공개 시스템의 0-day 또는 실증 익스플로잇 코드는 포함하지 않는다.
  • 본선 진출 시 NDA 정식 서명 후, 비공개 PoC·로그·취약점 정보는 별도 산출물로 통제 환경에서만 공유한다.
  • 대회 종료 후 1년간 비밀유지 의무를 준수하며, 외부 공개는 LIG D&A의 서면 동의 하에 추진한다.
  • 본 시스템은 방어(defensive) 목적으로 설계되었으며, 공격 시나리오는 평가용 시뮬레이션에 한정된다(MITRE ATT&CK의 평가 활용 원칙과 정합).
