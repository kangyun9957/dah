Executive Summary

본 보고서는 DAH 2026 예선 심사 배점표(공격 시나리오 30점, 방어 전략 25점, AI 에이전트 아키텍처 25점, 팀 역량 10점, 문서 완성도 10점)에 정합하도록 작성되었으며, "AI 에이전트가 자율적으로 판단하여 공방을 수행한다"는 본 대회의 본질적 차별점을 전제로 한다. 본선 채점식 total = (attack + defense) × availability는 단순한 침투/차단 능력이 아니라 "무력화 없이 통제력을 유지하는 능력"을 측정한다. 이는 군용 UAV/UGV 작전의 본질 — 임무 지속성(Mission Assurance) — 과 정확히 동일한 평가축이며, 본 보고서의 모든 설계 결정은 가용성 보존을 최우선 제약으로 둔다.

전제
UAV/UGV는 단순 IT가 아닌 CPS(Cyber-Physical System)이며, GNSS·RF·CAN·MAVLink·STANAG 4586·MIL-STD-1553 등 도메인 특화 표면을 갖는다.
접근
공격 10개 시나리오 / 방어 4-계층 / AI 에이전트 4-역할(Sentinel · Hunter · Healer · Coordinator). LLM(추론) + RL(제어) 하이브리드 의사결정.
성과 목표
공격 점수 ≥ 70% · 방어 차단율 ≥ 85% · SLA 가용성 ≥ 95% 유지로 가중 합계 최대화.

본 보고서의 4가지 차별 포인트

  1. 도메인 정합성 — MAVLink 2.0, STANAG 4586, DDS-Security(OMG), MIL-STD-1553B 등 실제 방산·무인이동체 프로토콜을 명시적으로 다룬다.
  2. AI-on-AI 위협 통합 — MITRE ATLAS 기반 적대적 머신러닝(Adversarial ML) 공격을 별도 시나리오가 아닌 모든 인지 계층의 횡단 위협으로 모델링한다.
  3. 가용성 우선 설계 — 채점식의 곱셈 항을 의식해 모든 대응 액션에 가용성 영향 비용 함수(c_avail)를 부여, 강화학습 보상 설계에 직접 반영한다.
  4. 증명 가능한 의사결정 — LLM 추론에는 Reasoning Trace + Tool Logs를 강제하여 사후 분석 및 NDA 준수성을 확보한다.

1. 서론 — UAV/UGV 사이버보안 패러다임

1.1 왜 "AI 공방전"인가

현대 무인이동체(UAV/UGV)는 통신·항법·제어·인지가 단일 임베디드 스택 내에서 결합되어 동작하는 사이버-물리 시스템이다. 공격 표면은 (1) RF/전자기 영역 — GNSS, 위성, 데이터링크, (2) 버스/프로토콜 영역 — CAN, MIL-STD-1553B, MAVLink, STANAG 4586, (3) 컴퓨트/OS 영역 — RTOS, Linux, 컨테이너, (4) 인지/AI 영역 — 비전·LiDAR·SLAM·강화학습 정책 으로 4중화되어 있다. 사람이 키보드로 이 4개 표면을 동시에 다루는 것은 비현실적이며, 실제 작전 템포에서는 판단 지연(latency) 자체가 패배 원인이다. DAH 2026이 사람이 아닌 AI 에이전트의 자율 공방을 요구하는 이유가 여기에 있다.

1.2 작전 환경의 제약 — 가용성이 곧 임무

본선 환경은 위성망(SATCOM)을 모사한 클라우드 인프라에서 동작한다. 위성 링크의 특성상 (i) 왕복 지연 600–800 ms, (ii) 대역폭 제한, (iii) 버스트 손실이 상시적이다. 방어 측 에이전트는 단순히 IDS 룰을 강화하는 것만으로는 임무 손실을 피할 수 없으며, "공격을 받는 중에도 비행체가 임무 궤적을 유지한다"는 자세 — 즉 회복탄력성(Resilience) — 이 핵심이다. 이는 NIST SP 800-160 Vol.2 "Developing Cyber Resilient Systems"의 원칙과 정합한다.

3. 공격 시나리오 설계 (예선 배점 30점)

본 절은 본선 공방을 가정한 10개 시나리오로 구성된다. 각 시나리오는 (1) 공격 목표 → (2) 표면/경로 → (3) AI 공격 에이전트 전략 → (4) 예상 방어 우회 → (5) 가용성 영향 평가의 5요소로 기술한다. 모든 시나리오는 방어 측 점수를 직접 깎거나, 가용성 계수를 직접 떨어뜨리도록 설계되었다.

S-1 · GNSS 스푸핑/재밍 (위성항법 기만)

목표
비행체 위치 추정을 점진적으로 이격(drag-off)시켜 임무 좌표 이탈 유도.
표면
L1/L2 GPS, Galileo E1/E5 RF 입력. 클라우드에서는 GNSS 시뮬레이션 토픽 변조.
AI 전략
RL 기반 점진적 위치 오프셋 — 비행체 INS 칼만필터의 잔차(residual) 임계를 초과하지 않으면서 좌표를 유도.
우회 포인트
Receiver Autonomous Integrity Monitoring(RAIM) · 단순 임계 IDS
가용성 영향
중(비행 지속, 좌표만 이탈). 채점 친화적.
참조
Humphreys(UT Austin) GNSS spoofing, NIST IR 8270 §4

S-2 · MAVLink 2.0 C2 하이재킹 (Adversary-in-the-Middle)

MAVLink는 기본 무인증 프로토콜이며, 2.0에서 서명 확장이 도입되었으나 키 회전·재생 보호가 약하다. 공격 에이전트는 위성 게이트웨이에서 (i) HEARTBEAT 시퀀스를 학습, (ii) 정상 트래픽 사이에 COMMAND_LONG 의 MAV_CMD_DO_REPOSITION·MAV_CMD_DO_SET_MODE 를 인젝션, (iii) STATUSTEXT를 위조하여 지상국 로그를 오염시킨다.

방어 우회: 서명 키가 PSK인 경우 LLM 에이전트가 트래픽 패턴 회귀로 키 회전 시점을 추정 → 회전 직전 윈도우에 인젝션 집중.

S-3 · 펌웨어/공급망 변조 (Supply-Chain Bootkit + ML Trojan)

  • Bootkit: Secure Boot 미적용 부트로더 영역에 부팅 시 메모리 패치를 삽입(MITRE T1542.003).
  • ML Trojan: 인지 모델에 트리거 패턴(특정 위장 텍스처)에 반응하는 백도어 삽입(ATLAS AML.T0018).
  • 대회 환경: 컨테이너 이미지 레지스트리 측에 변조된 layer 주입.

S-4 · 센서 적대적 입력 (LiDAR/Camera Adversarial)

경계 인식 모델의 출력을 변조하기 위해 (i) 카메라에 대해 적대적 패치(patch) 텍스처를 환경에 배치, (ii) LiDAR에 대해 가짜 점군을 모사 인젝션. 본 대회의 클라우드 환경에서는 시뮬레이션 시계열에 노이즈 토픽을 직접 발행한다.

S-5 · SATCOM MITM (위성 링크 중간자)

모사된 위성 게이트웨이 사이에 프록시를 설치, (i) STANAG 4586 LOI(Level of Interoperability) 메시지의 우선순위 필드를 변조, (ii) NTP 응답을 지연시켜 비행체 시간 동기를 흔든다. 시간 동기 흔들기는 후속 공격 S-2의 서명 재생 윈도우를 확장시키는 전제 공격.

S-6 · AI 모델 회피 — 정책망 적대적 입력

강화학습 기반 항로 결정망 π(s) 의 관측 s 에 적대적 노이즈를 가하여 선회 결정을 왜곡. 본 대회의 자율 회피 기동 모델이 표적. 공격 에이전트는 LLM으로 관측 토큰의 어떤 부위를 변조해야 가용성 영향 없이 결정만 흔드는지 추론한 후 RL로 미세조정한다.

S-7 · CAN / MIL-STD-1553B 메시지 인젝션 (UGV)

UGV의 구동·조향 명령은 CAN(J1939) 또는 1553B로 흐른다. RC(원격 단말)을 BC(Bus Controller)로 위장하는 Rogue BC(T0848)를 통해 가짜 조향각/스로틀 명령을 주기적으로 주입. 단발성이 아닌 저강도 지속(low-and-slow) 패턴으로 IDS 평균선을 회피.

S-8 · 스웜 합의 알고리즘 공격

다수 UAV의 합의(consensus) 기반 편대 비행에서 비잔틴 노드 1대가 거짓 위치·잔량 정보를 방송. 합의 알고리즘이 평균 기반인 경우 1/3 미만의 노드만 오염되어도 편대가 표적 영역을 이탈하도록 유도 가능.

S-9 · 사이드채널 키 추출

전력/전자기 분석으로 AES·ECDSA 키 추출. 대회 환경에서는 컨테이너 cgroup 카운터를 통한 timing 분석으로 모사.

S-10 · 자원 고갈 DoS (가용성 표적 — 고위험)

방어 측이 IDS 룰을 과도하게 적재했을 때를 노려 의도적 false positive 유발 트래픽으로 SIEM 큐를 폭주시킨다. 역설적으로 방어 측 자신의 가용성 계수를 떨어뜨리도록 유도하는 방어 자해(self-DoS) 유도형. 가장 채점 친화적 공격.

4. 방어 전략 (예선 배점 25점)

본 절은 Defense-in-Depth × Zero Trust × Resilient Control의 3축을 결합한 통합 방어 모델을 제시한다.

4.1 4-계층 Defense-in-Depth

┌────────── L4 인지/AI ───────────┐ 적대적 입력 탐지 · 모달리티 가중 · 정책망 검증(verified policy) │ (Vision, LiDAR, Policy π) │ ├────────── L3 컴퓨트/OS ──────────┤ Secure Boot · TPM · eBPF/Falco · Container sigstore │ (RTOS, Linux, Container) │ ├────────── L2 버스/프로토콜 ──────┤ MAVLink 서명 강화 · DDS-Security · 1553B 행위 IDS │ (MAVLink, DDS, CAN, 1553B) │ ├────────── L1 RF/EM ──────────────┤ GNSS 무결성(RAIM+) · 빔포밍 · Anti-Jam · Spoof 탐지 │ (GNSS, SATCOM, Data-Link) │ └──────────────────────────────────┘

L1 — RF/EM 계층

  • GNSS 무결성: 다중 위성군(GPS+Galileo+KASS) 교차검증, 클락 잔차 + 신호 강도(C/N0) + 도플러 일치성 기반 RAIM+.
  • Anti-Spoof: 단일 안테나 환경에서도 가능한 신호 도래각(AOA) 추정 + 동적 패턴 변화 감시.
  • SATCOM Anti-Jam: Frequency Hopping(STANAG 4606류) + 적응적 부호화율 변조.

L2 — 버스/프로토콜 계층

  • MAVLink 2.0 Hardening: HMAC-SHA256 서명 + 시퀀스 재생 보호 + 키 회전 주기 ≤ 30 s + timestamp 단조증가 강제.
  • DDS-Security(OMG): Authentication / Access Control / Cryptographic 3 플러그인 활성화, 토픽 단위 권한 분리.
  • 1553B 행위 IDS: BC가 주기적으로 발신하는 RT-RT 통신 패턴을 시계열 모델로 학습 → Rogue BC 탐지.
  • CAN J1939: PGN 단위 정상 시계열 학습, 이상 PGN 발생 시 Healer 에이전트에 즉시 보고.

L3 — 컴퓨트/OS 계층

  • Measured Boot + TPM: 부트체인 PCR 측정값 검증, Secure Boot 미통과 시 세이프모드 진입.
  • eBPF 기반 런타임 가시성: syscall · network · file 이벤트를 4 ms 단위로 수집(Tetragon 등 활용).
  • 컨테이너 무결성: cosign/sigstore 서명 검증, 변조된 layer 발견 시 컨테이너 격리.
  • 최소 권한: 미션 컨트롤 컨테이너는 CAP_NET_RAW 거부, 별도 게이트웨이 분리.

L4 — 인지/AI 계층

  • 적대적 입력 탐지: feature squeezing + Mahalanobis OOD 점수 + 모달리티 일관성(LiDAR↔Camera) 교차검증.
  • 정책망 안전망: 강화학습 π(s)의 행동 a를 별도의 Safety Shield(formally-verified rule)로 후필터링.
  • 모델 무결성: 모델 weights 해시 검증(매 부팅), MLOps 파이프라인의 데이터 lineage 보존.

4.2 Zero Trust for Tactical Edge

NIST SP 800-207 원칙을 무인이동체 환경에 적용한다. 핵심 원칙: "네트워크 위치는 신뢰의 근거가 아니다". 모든 MAVLink/DDS 메시지는 다음 5속성을 검사받는다 — 발신자 ID, 메시지 정책, 시간 무결성, 시퀀스 무결성, 컨텍스트(임무 단계). 정책엔진(PE)은 SPIFFE-style ID 토큰을 검증하고, 결정은 PEP(정책 시행점)에서 강제된다.

4.3 Resilient Control — 회복탄력성

방어가 실패해도 임무 가용성은 유지되어야 한다. 본 시스템은 다음 3 모드를 정의한다.

모드트리거행동가용성 영향
Normal모든 지표 정상풀 자율 비행100%
DegradedL1/L2 이상 감지임무 유지 + 검증 강화 + 보조 채널 우선≥ 95%
Safe-HoldL3/L4 위협 확정호버/정지 + 사전 검증된 복귀 좌표만 수신임무 일시 정지(가용성 60–80%)

4.4 방어 KPI

  • MTTD (탐지 평균시간) ≤ 2.5 초 (위성 RTT 고려)
  • MTTR (대응 평균시간) ≤ 5.0 초
  • 오탐율 (FPR) ≤ 1% (가용성 보존 최우선)
  • 임무 가용성 ≥ 95% 채점 윈도우 기준

5. AI 방어 에이전트 아키텍처 (예선 배점 25점)

5.1 설계 원칙

  1. 역할 분리: 단일 LLM 만능 에이전트가 아니라 4역할의 다중 에이전트. 인간 조직(SOC)을 모방.
  2. 이중 사고: System-1(빠른 RL 정책) + System-2(느린 LLM 추론) 하이브리드. Kahneman의 사고 모델 차용.
  3. 도구 사용: 모든 에이전트는 표준 도구 인터페이스(eBPF, Suricata, Falco, OpenZiti, MAVLink-proxy) 호출 가능.
  4. 설명가능성: 모든 결정은 Reasoning Trace를 남기며, 사후 NDA 감사 가능.
  5. 비용 함수 일치: 보상함수 R = ΔA + ΔD − λ·c_avail, λ는 채점식 곱셈 효과를 반영.

5.2 4역할 에이전트

Sentinel — 탐지 (Detection)
eBPF + Suricata + 인지 OOD 검출기에서 수집된 신호를 1차 분류. 경량 분류기(GBDT)와 시계열 이상치 모델(Isolation Forest, LSTM-VAE) 앙상블. 초당 수천 이벤트를 처리, 의심 토큰을 Hunter로 발행.
Hunter — 추론 · 조사 (Reasoning)
LLM 기반(예: Claude/Qwen). 의심 토큰 + 컨텍스트(임무 단계, 최근 명령, 인지 모달리티 불일치)를 받아 가설 → 검증 도구 호출 → 결론 의 ReAct 루프를 수행. Tool list: query_ebpf, diff_mavlink_signature, compare_lidar_camera 등.
Healer — 대응 · 복원 (Response)
RL 정책망 π_H(s). 행동공간 = {block, isolate, rollback, mode-shift, no-op}. 보상 R = ΔD − λc_avail 으로 학습 — 가용성을 손상시키지 않는 차단을 선호한다.
Coordinator — 조정 · 정책 (Policy)
임무 단계(이륙·순항·표적·복귀)별 정책 매트릭스 적용. 다른 3 에이전트의 결정 충돌을 조정. LLM 기반의 short-context 정책 검토.

5.4 Observation / Action Space

Healer RL 에이전트의 정의는 다음과 같다.

# Observation s_t
s_t = {
  rf:        {gnss_residual, cn0_var, sat_drift_rate},
  bus:       {mavlink_sig_fail_rate, dds_acl_deny_rate, can_anomaly_score},
  compute:   {syscall_kl_div, file_integrity_violations, cpu_throttle_ms},
  cognition: {lidar_camera_iou, ood_score, policy_entropy_delta},
  mission:   {phase, time_to_waypoint, fuel_pct},
  scoreboard:{availability_pct, attack_pts, defense_pts}
}

# Action a_t  (가용성 영향 비용 c_avail 부여)
A = [
  ("no_op",                  c_avail=0.00),
  ("rotate_mavlink_key",     c_avail=0.02),
  ("isolate_compromised_pod",c_avail=0.10),
  ("switch_modality_weight", c_avail=0.05),  # LiDAR vs Camera
  ("enter_safe_hold",        c_avail=0.35),
  ("rollback_firmware",      c_avail=0.45),
  ("kill_suspect_link",      c_avail=0.20),
]

# Reward — 채점식과 일치
R_t = (delta_attack + delta_defense) * (availability_t / 100.0)
      - lambda_a * c_avail(a_t)
      - lambda_f * false_positive_indicator

5.5 Hunter LLM의 도구 사용 패턴 (ReAct)

# Hunter의 단일 사이클 - 예: S-2 MAVLink 인젝션 의심
Thought: HEARTBEAT 시퀀스 누락 + COMMAND_LONG 빈도 급증. MITM 가능성 평가 필요.
Action:  query_ebpf(filter="mavsdk_proxy ingress 5s", fields="src,seq,sig_ok")
Obs:     56 packets, sig_ok=false: 31, src_anomaly: 22
Thought: 서명 실패율 55% > 임계 5%. S-2 강한 신호. 가용성 영향 최소 대응 탐색.
Action:  propose_action(candidates=["rotate_mavlink_key","kill_suspect_link"])
Obs:     rotate_key c_avail=0.02, kill_link c_avail=0.20
Thought: rotate_key 우선, 5초 내 회복 없을 시 kill_link 에스컬레이션.
Final:   handoff_to_healer(plan="rotate_key -> observe(5s) -> escalate")

5.6 학습 파이프라인

  • Pre-train: MITRE ATT&CK + ATLAS 시나리오 시뮬레이터에서 1000+ 에피소드.
  • Offline RL: Conservative Q-Learning(CQL)로 안전성 우선 정책 학습. 오프라인 데이터로 임무 정지 사고 회피.
  • Online RLHF/RLAIF: 시뮬레이션 내 인간 또는 상위 LLM의 행동 피드백.
  • Continual: 본선 환경 적응 시 Federated 방식(다른 비행체와 가중치 공유, raw 데이터 미전송).

5.7 안전 가드 — Safety Shield

RL 정책의 출력은 항상 Safety Shield를 통과한다. Shield는 formally-verified BTL(Boolean Temporal Logic) 규칙으로, 다음을 절대 위반하지 않는다 — (i) 비행체가 임무 영역을 1.5 km 이상 이탈하는 행동 금지, (ii) 동시 다발 격리(isolate) 3건 이상 금지(자해 방지), (iii) Safe-Hold는 Coordinator 동의 필수.

6. 채점식 최적화 — 가용성 보존 보상 설계

대회 채점식 S = (Attack + Defense) × Availability의 곱셈 항이 의미하는 바를 정량적으로 분해한다.

Attack = 70, Defense = 80, Avail = 100 → S = 150 × 1.00 = 150.0
Attack = 90, Defense = 95, Avail = 70 → S = 185 × 0.70 = 129.5
Attack = 60, Defense = 70, Avail = 100 → S = 130 × 1.00 = 130.0

즉, 같은 (A+D) 합계에서 가용성 30%pt 손실 = 점수 30% 손실. 본 시스템의 모든 액션은 c_avail를 부여받으며, Healer 정책은 가용성 보존 액션을 우선 채택하도록 학습된다. 대표적으로 "전체 차단" 대신 "키 회전 + 관찰 윈도우" 같은 저영향 대응 우선 패턴을 채택한다.

6.1 오탐 비용 곡선

오탐(False Positive)은 가용성 손실로 직결된다. λ_f 를 통한 학습 시 가용성 곡선:

Availability(%) 100 |█████████───── FPR=0.5% 95 |████████████───── FPR=1.0% ← 목표 85 |█████████████████───── FPR=3.0% 70 |█████████████████████ FPR=8.0% 0 1 3 5 8 FPR(%)

6.2 본선 라운드 전략

  1. 초반 5분: 공격은 S-1/S-5 같은 저영향 시나리오로 점수 누적, 방어는 baseline 가용성 100% 유지.
  2. 중반: S-2/S-7 본격 침투 시도. Healer는 키 회전 + 격리 1건 정도로 응대.
  3. 후반: S-10(자해 유도) 회피 — IDS 룰 임계를 동적으로 완화하여 큐 폭주 방지.
